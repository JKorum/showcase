Diet Helper

What Is It? ü§®

It is a web app dedicated to helping folks to improve their diet - the latest project I developed.

What Does It Do? üî•

Using the app, you can search for healthy recipes among many different cooking websites. You can configure the search by applying various filters, such as allergy constraints and calories range.

When you first visit the search page, the app will generate and fire a request based on a random combination of possible ingredients.

The app supports infinite scroll - it will resend your or a synthetic request automatically every time you reach the end of search results.

You can manage a collection gathered from recipes you liked.

Meal analysis is also possible. You can enter ingredients - the app will provide you with useful data, such as nutrients calories ratio.

I used the MERN stack in conjunction with Typescript. Edamam APIs - to fetch recipes data and nutrition analysis. Client-sessions for authentication with cookies. Plus, Bootstrap as a backbone for UI.

The main difficulty was to figure out how to properly use Typescript with third-party libraries. It took some time, but using a type system in the project reduced the number of possible problems radically.

Developer Network

What Is It? ü§î

It is a social network for programmers.

What Does It Do? üïπÔ∏è

The app can do quite a few things. Here I will list just some of them.

You can create a public profile and view other folks' profiles.

You can create, read, update, and delete posts and comments. You can filter and sort them based on different conditions.

When looking for developers based on their skill set, you can switch between different searching algorithms. In 'regular' mode, search results will include developers that have at least one of the skills specified by you. In 'strict' - developers must have all specified skills.

You will receive a welcome email once registered, and a farewell when decided to delete the account. If you have no active sessions, you will be receiving emails when someone commented on your post.

You will get helper messages from the app alert system.

You can switch between gravatar and avatar images at any time - all your previous posts and comments will reflect this change.

Main Tools üß∞

I used the MERN stack for this one. Also, I used Redux, Redux Thunk, and React hooks to handle global and local app states. SendGrid API - to set up an email notification system. React Router - to manage client-side routing. JSON Web Token for authentication. Plus, SCSS - to style the app and make the layout responsive.

Challenges üòì

Frankly, there were a lot of them.

I think the main hurdle was to synchronize data between different system components correctly. Plus, I struggled with the problem of unexpected nullish and undefined values on the front-end.

Besides that, it was not easy to implement the idea of avatars switching. I had to alter existing Mongoose schemas radically. This alteration led to the need for many modifications in the codebase.
